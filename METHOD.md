# METHOD: Harness Engineering для Phase 1

## Принцип
**Humans steer, agents execute.**

Человек задаёт цель, ограничения, критерии качества и риск-политику.
Агент выполняет изменения, запускает проверки, обновляет документацию и оставляет прозрачный след решений.

## Рабочий цикл
1. Определить задачу и ожидаемый результат.
2. Проверить ограничения из `docs/design-docs/rules.md`.
3. Выполнить изменение минимальным шагом.
4. Запустить lint/build/test и зафиксировать результаты.
5. Обновить документацию и TODO.
6. Передать на review человеку.
