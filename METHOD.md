# METHOD: Harness Engineering в шаблоне

## Принцип
**Humans steer, agents execute**: человек проектирует среду, ограничения и контуры проверки; агент исполняет задачи и закрывает цикл проверок.

## Рабочий цикл
1. Человек задаёт цель и ограничения.
2. Агент строит/обновляет решение через короткий PR.
3. Агент запускает lint/build/test и доп. проверки.
4. Агент обновляет docs и TODO-реестр.
5. CI подтверждает, что код и документация согласованы.

## Обязательные свойства scaffold
- Legibility: агент может запускать проект и проверки.
- Guardrails: есть custom linters, structural tests, dependency rules.
- Документы в репозитории и версионируются вместе с кодом.
- Энтропия управляется регулярным doc gardening и tech-debt tracker.
