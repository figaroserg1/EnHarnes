# METHOD — краткая памятка по подходу

## 1) Сначала контекст, потом код
- Сначала заполняем `system-description` и `rules`.
- Только после этого проектируем архитектуру и пишем скрипты.

## 2) Если не знаем — не выдумываем
Пишем:

```txt
TODO: <что именно нужно описать>
Owner: AI->HUMAN
Status: BLOCKED (нужны ответы человека)
Example (REPLACE ME): <пример-заглушка>
```

## 3) Фиксируем ответственность
У каждого TODO должен быть владелец:
- `HUMAN` — должен написать человек;
- `AI` — ИИ может заполнить сам на основе доступного контекста;
- `AI->HUMAN` — ИИ обязан сначала опросить человека (предпочтительно).

## 4) Минимально рабочие заготовки > пустых файлов
Для линтера/сборки оставляем рабочий baseline, который легко расширить.

## 5) Обновляем референсы
В `docs/references.md` добавляем внешние ссылки на проверенные источники и примеры.
