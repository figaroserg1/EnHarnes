# AGENTS.md

# Карта репозитория для агентов

**ВНИМАНИЕ АГЕНТУ:** Этот файл — навигационная карта. Единственный источник истины — документы в `docs/`. Ничего не придумывай вне них.

## Основные правила работы
- Всегда соблюдай однонаправленную архитектуру → `ARCHITECTURE.md`
- Никогда не пиши код без утверждённого ExecPlan → `docs/PLANS.md`
- Валидируй данные на границах слоёв, без "YOLO-парсинга"
- Используй существующие утилиты, не создавай дубликаты
- Перед завершением сессии обновляй `progress.txt`

## Структура проекта
- `README.md`, `QUICKSTART.md`, `METHOD.md` — стартовые документы
- `ARCHITECTURE.md` — high-level архитектура
- `docs/design-docs/` — правила, системные решения, архитектурные спецификации
- `docs/product-specs/` — продуктовые требования
- `docs/exec-plans/` — активные и завершённые ExecPlans
- `docs/references/` — справочные материалы по библиотекам
- `docs/generated/db-schema.md` — актуальная схема БД (обновлять при миграциях)
- `scripts/` — операционные команды
- `tools/` — инфраструктурные и QA-утилиты

## Где искать информацию
- Общий контекст → `README.md`, `ARCHITECTURE.md`, `METHOD.md`
- Правила разработки → `docs/design-docs/rules.md`
- Frontend правила → `docs/FRONTEND.md`
- Планирование работ → `docs/PLANS.md`, `docs/exec-plans/`
- Надёжность и безопасность →  
  `docs/RELIABILITY.md`  
  `docs/SECURITY.md`  
  `docs/OBSERVABILITY.md`

## ExecPlans
Любая сложная задача или рефакторинг выполняется через ExecPlan  
(см. `docs/PLANS.md`) от дизайна до имплементации.
Код допускается только после появления плана в `docs/exec-plans/active/`.
