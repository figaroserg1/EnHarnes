# Порядок заполнения TODO (пошагово)

## Шаг 0. Клонирование и старт
1. Создай репозиторий из template.
2. Проверь, что все файлы шаблона на месте.
3. Создай ветку `setup/project-bootstrap`.

## Шаг 1. Описание системы (`docs/system-description.md`)
**Цель:** определить, что вообще строим.

Заполнять в порядке:
1. Scope и границы системы.
2. Пользователи/акторы.
3. Бизнес-цели и не-цели.
4. Ограничения (бюджет, сроки, compliance).

Сначала закрыть все TODO с `Priority: P0`.

## Шаг 2. Правила и политики (`docs/rules.md`)
**Цель:** зафиксировать, как можно/нельзя работать.

Порядок:
1. Security и доступы.
2. Data handling / privacy.
3. Engineering conventions.
4. Release и incident-процесс.

## Шаг 3. Архитектура (`docs/architecture.md`)
**Цель:** выбрать технический путь.

Порядок:
1. Контекстная диаграмма и ключевые компоненты.
2. Критичные решения (ADR-style).
3. NFR: performance, reliability, observability.
4. Риски и trade-offs.

## Шаг 4. Инструменты агента (`tools/setup-agent-tools.sh`)
**Цель:** дать воспроизводимый setup для skills/MCP.

Порядок:
1. Внести конкретные команды установки под вашу среду.
2. Вписать нужные MCP endpoints.
3. Проверить команды на чистом окружении.

## Шаг 5. Скрипты проекта (`scripts/*.sh`)
**Цель:** чтобы репо сразу было "живым".

Порядок:
1. Кастомный линтер (`custom-linter.sh`) — добавить доменные проверки.
2. Скрипт сборки (`build.sh`) — подключить реальные команды сборки/тестов.

## Шаг 6. Финальная валидация
- Запустить `bash scripts/custom-linter.sh`.
- Запустить `bash scripts/build.sh`.
- Проверить, что не осталось критичных TODO (`P0`) без owner/status.
