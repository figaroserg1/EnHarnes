# EnHarness — универсальный scaffold для проектных репозиториев

Этот репозиторий — **шаблон**, который помогает быстро стартовать проект, даже если пока нет полной ясности по системе, правилам, архитектуре и процессам.

> ⚠️ Важный принцип шаблона: если информации пока нет, оставляем `TODO:` + пометку, **кто должен заполнить**:
>
> - `Owner: HUMAN`
> - `Owner: AI`
> - `Owner: AI->HUMAN` (предпочтительный вариант: ИИ задаёт вопросы человеку и заполняет по ответам)

## Что внутри

- `METHOD.md` — короткая памятка по подходу.
- `docs/TODO_FILL_ORDER.md` — пошаговый порядок, в каком порядке заполнять TODO.
- `docs/system-description.md` — шаблон описания системы.
- `docs/rules.md` — шаблон правил/политик/ограничений.
- `docs/architecture.md` — шаблон архитектуры + ссылки на внешние примеры.
- `docs/references.md` — полезные внешние гайды и референсы.
- `scripts/custom-linter.sh` — заготовка кастомного линтера (минимально рабочая).
- `scripts/build.sh` — заготовка сборки/проверок (минимально рабочая).
- `tools/setup-agent-tools.sh` — команды установки skills/MCP для агента (редактируется под команду).

## Быстрый старт

1. Открой `docs/TODO_FILL_ORDER.md` и пройди шаги по порядку.
2. Заполни обязательные TODO в `docs/system-description.md` и `docs/rules.md`.
3. Зафиксируй архитектурные решения в `docs/architecture.md`.
4. Настрой инструменты агента в `tools/setup-agent-tools.sh`.
5. Запусти базовые проверки:
   - `bash scripts/custom-linter.sh`
   - `bash scripts/build.sh`

## Как использовать с ИИ

Рекомендуемый режим:

1. ИИ читает TODO с пометкой `Owner: AI->HUMAN`.
2. ИИ задаёт короткие уточняющие вопросы человеку.
3. После ответов ИИ заполняет соответствующие разделы.
4. Человек утверждает результат PR-ревью.

## Публикация

Это публичный template/scaffold для GitHub. Рекомендуется:

- включить `Use this template` в настройках репозитория;
- поддерживать changelog шаблона;
- добавлять новые секции только с явными TODO и owner-метками.
