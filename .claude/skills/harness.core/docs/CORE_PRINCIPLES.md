# Core Beliefs

Operational facts that govern how agents and humans work in this repository.

## 1. Repository Is the Single Source of Truth

If it is not committed to this repo, it does not exist to the agent.
No Slack threads, no verbal agreements, no Google Docs.
Every decision, constraint, and plan lives in versioned markdown.

## 2. Agents Are First-Class Team Members

Documentation and architecture are written for agent consumption first.
Human readability is a bonus, not the primary goal.
If an agent cannot act on a document without guessing, the document is broken.

## 3. Corrections Are Cheap. Blocking Is Expensive.

In a high-throughput agent workflow, fixing forward costs less than waiting for perfect.
Minimal CI gates. No flaky-test blocking. Unresolved review threads are the only hard block.

## 4. Constraints Enable Speed

Strict layered architecture, enforced mechanically by linters and structural tests,
allows the agent to ship fast without architectural drift. Freedom exists within boundaries.
Each project defines its own layers in `policies/architecture.yaml` — do NOT assume a fixed set.
Example layers: `Types → Config → Repo → Service → Runtime → UI` (illustrative only).

## 5. Taste Is Captured Once, Enforced Continuously

Human judgment about structure, naming, and logging style is encoded in
`docs/harness/GOLDEN_PRINCIPLES.md` and executed by automated linters — not by code review.
Once written, it applies to every line generated by every agent session.

## 6. Technical Debt Is a High-Interest Loan

Debt is paid in small continuous increments via entropy checks, not in painful
quarterly bursts. Background agents run scheduled cleanup. See `docs/harness/ENTROPY_PRINCIPLES.md`.

## 7. Progressive Disclosure Over Information Dumps

Agents start from `AGENTS.md` (a map, ~100 lines).
Deeper context lives in `docs/` and is fetched when needed.
Do not put everything in one file.
